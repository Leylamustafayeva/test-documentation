# User Stories & Acceptance Criteria

---

## 🟢 User Story 1: Məhsul ölçülərinin göstərilməsi

*User Story:*  
Mən istifadəçi olaraq, məhsulun üzərinə klik etdikdə məhsulun mövcud ölçülərini (sizes) görmək istəyirəm ki, mənə uyğun ölçünü rahatlıqla seçə bilim.

*Acceptance Criteria:*
1. İstifadəçi məhsulun üzərinə klik etdikdə, məhsul haqqında detallı məlumatlar göstərilən səhifə açılmalıdır.  
2. Məhsula uyğun bütün mövcud ölçülər həmin səhifədə öz əksini tapmalıdır.  
3. Mövcud olmayan ölçülər deaktiv (disabled) vəziyyətdə göstərilməlidir.  
4. İstifadəçi ölçü seçməzdən öncə “Add to cart” düyməsi aktiv olmamalıdır.  
5. Seçilmiş ölçü vizual olaraq digər ölçülərdən fərqlənməlidir.

## 🟢 User Story 2: Filtr funksiyası

*User Story:*  
Mən istifadəçi olaraq, məhsulları müxtəlif aspektlərdən (material, qiymət və s.) filtr etmək istəyirəm ki, axtardığım məhsulu tez tapım və vaxta qənaət edim.

*Acceptance Criteria:*
1. Filtr düyməsi saytda əlçatan və görünən yerdə yerləşdirilməlidir.  
2. İstifadəçi məhsulları seçilmiş xüsusiyyətlərə görə asanlıqla filtr edə bilməlidir.  
3. İstifadəçi eyni anda bir neçə filtr seçə bilməlidir.  
4. Filtrə uyğun məhsullar səhifədə düzgün şəkildə göstərilməlidir.  
5. Filtrləri sıfırlamaq (reset) mümkün olmalıdır.

## 🟢 User Story 3: WhatsApp vasitəsilə əlaqə

*User Story:*  
Mən istifadəçi olaraq, saytda WhatsApp logosuna klik etdikdə birbaşa WhatsApp-a yönləndirilmək istəyirəm ki, sayt sahibi ilə tez və rahat əlaqə saxlaya bilim.

*Acceptance Criteria:*
1. Saytın uyğun və görünən hissəsində WhatsApp logosu yerləşdirilməlidir.  
2. Logo kliklənə bilən (clickable) olmalıdır.  
3. Klik etdikdə istifadəçi birbaşa WhatsApp-a yönləndirilməlidir.  
4. Mobil cihazlarda WhatsApp tətbiqi açılmalıdır.  
5. Desktop mühitində WhatsApp Web və ya tətbiqə yönləndirmə düzgün işləməlidir.  
6. Yönləndirilən WhatsApp hesabı sayt sahibinə məxsus olmalıdır.
